Guía de instalación WRF model

Guía instalacion por Lainer

Junio 04, 2024

El modelo WRF, que significa Weather Research and Forecasting, es un avanzado sistema de modelado atmosférico desarrollado para simular y predecir el comportamiento del tiempo en la atmósfera terrestre. Este modelo es ampliamente utilizado en la investigación meteorológica y en la predicción del tiempo a corto y mediano plazo.

Desarrollado inicialmente por el Centro Nacional de Investigación Atmosférica (NCAR) y la Administración Nacional Oceánica y Atmosférica (NOAA) en los Estados Unidos, el WRF ha evolucionado para convertirse en un modelo flexible y de código abierto. Su flexibilidad permite adaptarse a una variedad de escalas espaciales y temporales, lo que lo hace adecuado para estudiar fenómenos atmosféricos locales y globales.

El WRF utiliza técnicas de integración numérica avanzadas para simular procesos atmosféricos complejos, como la convección, la radiación, la interacción suelo-atmósfera y otros procesos físicos esenciales. Además, el modelo puede ser configurado para simular diversos aspectos meteorológicos, como la temperatura, la humedad, la velocidad del viento y las precipitaciones.

Debido a su capacidad para proporcionar pronósticos meteorológicos detallados y personalizados, el modelo WRF es utilizado por meteorólogos, investigadores y científicos en todo el mundo para entender y prever el tiempo con mayor precisión. Su versatilidad y aplicaciones extensas hacen del WRF un instrumento valioso en la investigación climática y en la mejora de las capacidades de pronóstico meteorológico.

En esta guía se describe como instalar el WRF en los siguientes pasos:

  • Pruebas del entorno del sistema.
  • Instalación de las bibliotecas de construcción necesaria para compilar WRF.
  • Pruebas de compatibilidad de bibliotecas.
  • Compilar el WRF.
  • Compilar WPS.
  • Datos de geografía estática.

Pruebas del entorno del sistema.

  1. En primer lugar, es muy importante tener un compilador gfortran, además de gcc y cpp. Para probar si existen en el sistema, escriba lo siguiente:
    • which gfortran
    • which cpp
    • which gcc

    Si los tiene instalados, se le debe proporcionar una ruta para la ubicación de cada uno.

    Se recomienda un compilador gcc versión 1, en prueba reciente se demostró que es una de las versiones que funciona de mejor manera para compilar la versión de WRF 4.5. Para determinar que versión de gcc tiene instalado ejecute:

    • gcc –version o gcc -v
  2. Crear una área de trabajo para la instalación de software. Una carpeta que donde se encontrará todos los ficheros y directorios necesarios. Ejemplo Build_WRF. Dentro de esta carpeta debería contener las carpetas: LIBRARIES, TESTS, WRF y WPS

  3. Hay algunas pruebas simples que se pueden ejecutar para verificar que el compilador Fortran esté construido correctamente y que sea compatible con el compilador C.

    NOTA: Si alguna de estas pruebas falla, deberá comunicarse con el administrador de sistemas de su institución para obtener ayuda, ya que son específicas de su entorno particular y no tenemos los recursos para respaldar este tipo de errores.

    A continuación se muestra un archivo tar que contiene las pruebas. Descargue el archivo tar y colóquelo en el directorio TESTS.

    Fortran_C_tests.tar

    Para descomprimir el archivo tar, escriba:

    • tar -xf Fortran_C_tests.tar

    o

    • tar -xvf Fortran_C_tests.tar(Esta opción permite estar más en control)

    Hay 7 pruebas disponibles, para comenzar desde arriba y realizarlas, una a la vez.

    Prueba #1: Prueba Fortran de formato fijo: TEST_1_fortran_only_fixed.f

    Escriba lo siguiente en la línea de comando:

    • gfortran TEST_1_fortran_only_fixed.f

    Ahora escribe:

    • ./a.out

    Lo siguiente debería imprimirse en la pantalla:

    SUCCESS test 1 fortran only fixed format

    Prueba #2: Fortran de formato libre: TEST_2_fortran_only_free.f90

    Escriba lo siguiente en la línea de comando:

    • gfortran TEST_2_fortran_only_free.f90

    Ahora escribe:

    • ./a.out

    Lo siguiente debería imprimirse en la pantalla: 

                    
                  Assume Fortran 2003: has FLUSH, ALLOCATABLE, derived 
                  type, and ISO C Bindin
                  SUCCESS test 2 fortran only free format

    Prueba #3: C: TEST_3_c_only.c

    Escriba lo siguiente en la línea de comando:

    • gcc TEST_3_c_only.c

    Ahora escribe:

    • ./a.out

    Lo siguiente debería imprimirse en la pantalla:

    
                SUCCESS test 3 c only

    Prueba #4: Fortran llamando a una función C (nuestro gcc y gfortran tienen diferentes valores predeterminados, por lo que obligamos a ambos a usar siempre 64 bits [-m64] al combinarlos): TEST_4_fortran+c_c.c, y TEST_4_fortran+x_f.f90

    Escriba lo siguiente en la línea de comando:

    • gcc -c -m64 TEST_4_fortran+c_c.c

    y escribe después:

    • gfortran -c -m64 TEST_4_fortran+c_f.f90

    luego:

    • gfortran -m64 TEST_4_fortran+c_f.o TEST_4_fortran+c_c.o

    posteriormente escribe:

    • ./a.out

    Lo siguiente debería imprimirse en la pantalla:

    
                C function called by Fortran
                Values are xx = 2.00 and ii = 1
                SUCCESS test 4 fortran calling c

  4. Además de los compiladores necesarios para fabricar los ejecutables de WRF, el sistema de compilación de WRF tiene scripts como nivel superior para la interfaz de usuario. El sistema de secuencias de comandos WRF utiliza y, por lo tanto, es necesario tener lo siguiente:

    • csh
    • perl
    • sh

    Para probar si estos lenguajes de secuencias de comandos funcionan correctamente en el sistema, hay que ejecutar 3 pruebas. Estas pruebas se incluyeron en el "Archivo Tar de pruebas Fortran y C".

    Prueba #5: csh En la línea de comando, escriba:

    • ./TEST_csh.csh

    el resultado debería ser:

    SUCCESS csh test

    Prueba #6: perl En la línea de comando, escriba:

    • ./TEST_perl.pl

    el resultado debería ser:

    SUCCESS perl test

    Prueba #7: sh En la línea de comando, escriba:

    • ./TEST_sh.sh

    el resultado debería ser:

    SUCCESS sh test

Instalación de las bibliotecas de construcción necesaria para compilar WRF.

Antes de comenzar, necesita crear otro directorio. Vaya a su directorio Build_WRF:

  • cd Build_WRF

y luego crear un directorio llamado "LIBRARIES"

  • mkdir LIBRARIES

Dependiendo del tipo de ejecución que desee realizar, existen varias bibliotecas que deben instalarse. A continuación se muestran 5 bibliotecas. Descargue los 5 archivos tar y colóquelos en el directorio LIBRARIES.

  • netcdf-c-4.7.2
  • netcdf-fortran-4.5.2
  • Mpich-4.1.2
  • zlib-1.2.11
  • Libpng-1.2.50
  • Jasper-1.900.1

Es importante tener en cuenta que todas estas bibliotecas deben instalarse con los mismos compiladores que se utilizarán para instalar WRF y WPS.

Antes de instalar las bibliotecas, es necesario configurar estas rutas con vim /home/user/.bashrc: 

export WRFDIR=/home/user/Build_WRF/LIBRARIES
export CC=gcc
export CXX=g++
export FC=gfortran
export FCFLAGS=-m64
export F77=gfortran
export FFLAGS=-m64
export JASPERLIB=$WRFDIR/grib2/lib
export JASPERINC=$WRFDIR/grib2/include
export LDFLAGS=-L$WRFDIR/grib2/lib
export CPPFLAGS=-I$WRFDIR/grib2/include

export PATH=$WRFDIR/netcdf/bin:$PATH
export NETCDF=$WRFDIR/netcdf
export CPPFLAGS=-I$WRFDIR/netcdf/include
export LDFLAGS=-L$WRFDIR/netcdf/lib

export PATH=$WRFDIR/mpich/bin:$PATH

También es importante actualizar las bashrc con el comando:

  • source /home/user/.bashrc
  1. NetCDF: ¡Esta biblioteca siempre es necesaria! Esta instalación consta de una biblioteca netcdf-c y netcdf-fortran.
    • tar xzvf netcdf-c-4.7.2.tar.gz
    • cd netcdf-c-4.7.2
    • ./configure --prefix=$DIR/netcdf --disable-dap --disable-netcdf-4 --disable-shared
    • make
    • make install
    • cd .. (Añadir el set netcdf c al bashrc)
    • tar xzvf netcdf-fortran-4.5.2.tar.gz
    • cd netcdf-fortran-4.5.2
    • ./configure --prefix=$DIR/netcdf --disable-dap --disable-netcdf-4 --disable-shared
    • make
    • make install
    • cd ..

  2. MPICH: esta biblioteca es necesaria si planea construir WRF en paralelo. Si su máquina no tiene más de 1 procesador, o si no necesita ejecutar WRF con múltiples procesadores, puede omitir la instalación de MPICH.

    En principio, cualquier implementación del estándar MPI-2 debería funcionar con WRF; sin embargo, tenemos la mayor experiencia con MPICH y, por lo tanto, eso es lo que se describirá aquí.

    Suponiendo que todos los comandos 'export' ya se emitieron al configurar NetCDF, puede continuar con la instalación de MPICH, emitiendo cada uno de los siguientes comandos:

    • tar xzvf mpich-3.0.4.tar.gz
    • cd mpich-3.0.4
    • ./configure --prefix=$DIR/mpich
    • make
    • make install
    • cd ..

  3. zlib: esta es una biblioteca de compresión necesaria para compilar WPS (específicamente ungrib) con capacidad GRIB2.

    Suponiendo que todos los comandos "export" de la instalación de NetCDF ya estén configurados, puede pasar a los comandos para instalar zlib.

    • tar xzvf zlib-1.2.7.tar.gz cd zlib-1.2.7
    • ./configure --prefix=$DIR/grib2
    • make
    • make install
    • cd ..

  4. libpng: esta es una biblioteca de compresión necesaria para compilar WPS (específicamente ungrib) con capacidad GRIB2.

    Suponiendo que todos los comandos "export" de la instalación de NetCDF ya estén configurados, puede pasar a los comandos para instalar zlib.

    • tar xzvf libpng-1.2.50.tar.gz
    • cd libpng-1.2.50
    • ./configure --prefix=$DIR/grib2
    • make
    • make install
    • cd ..

  5. Jasper: esta es una biblioteca de compresión necesaria para compilar WPS (específicamente ungrib) con capacidad GRIB2.

    Suponiendo que todos los comandos "export" de la instalación de NetCDF ya estén configurados, puede pasar a los comandos para instalar zlib.

    • tar xzvf jasper-1.900.1.tar.gz
    • cd jasper-1.900.1
    • ./configure --prefix=$DIR/grib2
    • make
    • make install
    • cd ..

Pruebas de compatibilidad de bibliotecas.

Una vez que la máquina de destino sea capaz de crear pequeños ejecutables de Fortran y C (lo que se verificó en la sección Pruebas del entorno del sistema), y después de que se hayan construido las bibliotecas NetCDF y MPI (dos de las bibliotecas de la sección Creación de bibliotecas), emular el WRF. Comportamiento del código, se requieren dos pequeñas pruebas adicionales. Necesitamos verificar que las bibliotecas puedan funcionar con los compiladores que se utilizarán para las compilaciones WPS y WRF.

NOTA: Si alguna de estas pruebas falla, deberá comunicarse con el administrador de sistemas de su institución para obtener ayuda, ya que son específicas de su entorno particular y no tenemos los recursos para respaldar este tipo de errores.

A continuación se muestra un archivo tar que contiene estas pruebas. Descargue este archivo tar y colóquelo en el directorio TESTS, y luego "cd" en el directorio TESTS:

Fortran_C_NETCDF_MPI_tests.tar

Para descomprimir el archivo tar, escriba:

  • tar -xf Fortran_C_NETCDF_MPI_tests.tar

Hay 2 pruebas:

  1. Prueba #1: Fortran + C + NetCDF

    La prueba exclusiva de NetCDF requiere que el archivo de inclusión del paquete NETCDF esté en este directorio. Copie el archivo aquí:

    • cp ${NETCDF}/include/netcdf.inc .

    Compile los códigos Fortran y C para esta prueba (la opción -c indica que no se debe intentar crear un ejecutable). Escriba los siguientes comandos:

    • gfortran -c 01_fortran+c+netcdf_f.f
    • gcc -c 01_fortran+c+netcdf_c.c
    • gfortran 01_fortran+c+netcdf_f.o 01_fortran+c+netcdf_c.o \ -L${NETCDF}/lib -lnetcdff -lnetcdf
    • ./a.out

    Lo siguiente debería aparecer en su pantalla:

    
C function called by Fortran
Values are xx = 2.00 and ii = 1
SUCCESS test 1 fortran + c + netcdf

  2. Prueba #2: Fortran + C + NetCDF + MPI

    La prueba NetCDF+MPI requiere que los archivos de inclusión de ambos paquetes estén en este directorio, pero los scripts MPI automáticamente hacen que el archivo mpif.h esté disponible sin ayuda, por lo que no es necesario copiarlo. Copie el archivo de inclusión de NetCDF aquí:

    • cp ${NETCDF}/include/netcdf.inc .

    Tenga en cuenta que los ejecutables MPI mpif90 y mpicc se utilizan a continuación al compilar. Emita los siguientes comandos:

    • mpif90 -c 02_fortran+c+netcdf+mpi_f.f
    • mpicc -c 02_fortran+c+netcdf+mpi_c.c
    • mpif90 02_fortran+c+netcdf+mpi_f.o \ 02_fortran+c+netcdf+mpi_c.o \ -L${NETCDF}/lib -lnetcdff -lnetcdf
    • mpirun ./a.out

    Lo siguiente debería aparecer en su pantalla: 

    
C function called by Fortran
Values are xx = 2.00 and ii = 1
status = 2
SUCCESS test 2 fortran + c + netcdf + mpi
